專案目標(111310553 資工二 王淳禧)

從最基礎的 Nand 邏輯閘開始，逐層建構出 16 位元的 Hack Computer 系統。本階段完成從硬體邏輯閘（HDL）到簡易機器語言程式（ASM）的整合。

實作內容說明
Chapter 1: 
  基礎邏輯閘 (Boolean Logic)
  實作重點：僅使用 Nand 組合出 Not、And、Or、Xor 等基礎閘。

關鍵零件：

Mux (多路選擇器)：根據選擇訊號 sel 來決定輸出 A 或 B，這是後續 CPU 切換資料來源的核心零件。

DMux (解多路選擇器)：將輸入訊號分流到指定位置。

Chapter 2: 
  算術運算單元 (ALU)
  實作重點：建立加法器與 ALU。

邏輯架構：
  從 HalfAdder（半加器）與 FullAdder（全加器）組成 Add16。
  ALU 設計：透過 6 個控制位元對輸入進行處理。這部分最重要的是輸出 zr (判斷是否為 0) 與 ng (判斷是否為負數)，這兩個數值直接影響到第五章的條件跳轉邏輯。

Chapter 3: 
  記憶體系統 (Registers & RAM)
  實作重點：引入時脈訊號，讓電路具備儲存功能。

階層結構：

利用 DFF 製作出 Bit，再組成 16-bit 的 Register。

RAM 擴充：將 Register 組合為 RAM8，並依此邏輯擴展至 RAM16K。

PC (程式計數器)：負責管理指令地址，支援重設 (Reset)、遞增 (Inc) 與載入 (Load) 功能。

Chapter 4: 機器語言 (Assembly)
實作重點：使用 Hack 組合語言編寫程式，驗證硬體邏輯。

練習項目：

Mult.asm：透過迴圈與加法實作乘法運算。

Fill.asm：藉由監測鍵盤輸入（KBD），直接改寫螢幕記憶體（SCREEN）的數值，達成黑白畫面切換。

Chapter 5: 電腦架構整合 (Computer Architecture)
實作重點：將前四章的元件整合成完整的 CPU 與 Computer。

關鍵邏輯：

CPU 指令解碼：區分 A 指令（常數載入）與 C 指令（運算控制）。C 指令需拆解出 ALU 控制碼、目的地（Dest）以及跳轉條件（Jump）。

Memory Map：將 RAM、Screen 與 Keyboard 映射到同一個定址空間，方便 CPU 進行 I/O 操作。

實作心得與改善點
零件復用性：在實作過程中，發現前一章的零件穩定度非常重要（例如 Mux16）。若底層零件有 bug，高層架構會完全無法運作。

指令解碼優化：在 CPU.hdl 中，處理 C 指令的控制位元時，應先確認 Instruction[15] 是否為 1，避免 A 指令錯誤觸發寫入訊號。

Debug 流程：優先使用硬體模擬器跑測試腳本（.tst），確認輸出波形符合預期，這比直接看程式碼檢查有效率。
